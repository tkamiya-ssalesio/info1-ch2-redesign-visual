## 🛠 実習の手順

### Step 1．解説ページで情報デザインの原則を体験しよう
1.  Codespacesの画面右下から **Live Server** を起動し，[lecture.html](file:///Users/kamiyatoshiki/Documents/GitHub/info1-ch2-redesign-visual/lecture.html) をブラウザで開きます．
2.  実際の画面を見ながら，以下の「情報デザインの4大原則」を体感し，デモページの解説をよく読みましょう．
    *   **可読性・視認性・判読性：** 読みやすいフォントの選定と，見出しと本文の強弱（対比）．
    *   **配色とコントラスト：** 全体を4色以内に抑えることと，背景と文字の十分な明度差（コントラスト比4.5:1以上）．
    *   **カラーユニバーサルデザイン：** 色だけに頼らず，太字，下線，枠線，記号（⚠️など）を併用する強調．
    *   **レイアウトと余白：** 関連する情報を近づける（近接）ことと，左端をきれいに揃えて視線を流す（整列）こと．

### Step 2．演習用のHTML構造を修正しよう（構造化のデバッグ）
1.  [index.html](file:///Users/kamiyatoshiki/Documents/GitHub/info1-ch2-redesign-visual/index.html) を開きます．
2.  CSSでデザインを当てる前提として，不適切なHTML構造（見出しレベル `h1` と `h2` の順序逆転，表内の不要な `<br>`，リストタグのエラーなど）を正しい論理構造にデバッグします．
3.  必要に応じて，特定の要素をCSSで指定しやすくするためにクラス名（`class="xxx"`）を追加します．

### Step 3．最悪な初期デザインの CSS をレスキューしよう
1.  [style.css](file:///Users/kamiyatoshiki/Documents/GitHub/info1-ch2-redesign-visual/style.css) を開きます．
2.  現在適用されている「目がチカチカするスタイル」の数値や色コードを書き換えて，`lecture.html` で学んだ原則を満たすように美しくリデザインします．
    *   **配色:** 背景色，基本文字色，メインカラー，強調色の4色以内に整理します．
    *   **フォント:** 読みやすいゴシック体に指定し，行間（`line-height: 1.6`）と，見出しのフォントサイズ（ジャンプ率）を調整します．
    *   **アクセシビリティ:** 重要な連絡先や警告部分に，色（赤など）以外の手がかり（太字，枠線，絵文字等）を併用します．
    *   **余白:** 各コンテンツの間に適切な外側余白（`margin`）を空け，テーブルのセル内側に適切な余白（`padding: 8px`〜`12px`）を指定して読みやすく整えます．

### Step 4．Gitによる提出と自動評価
1.  AIとのチャット履歴を `copilot-chat-history.json` としてエクスポートします．
2.  ファイルを保存し，Gitの3ステップで提出します：
    ```bash
    git add .
    git commit -m "feat: style index.html"
    git push origin main
    ```
3.  GitHubの「Issues」タブを開き，約1分後に届くAI先生からの自動フィードバックを確認します．赤いバツ（🔴）が出た場合は，アドバイスを読んでコードを修正し，再度コミット＆プッシュしてください．すべて🟢（合格）になるまで繰り返します！
